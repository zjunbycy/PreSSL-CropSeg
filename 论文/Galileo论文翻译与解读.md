## 摘要（Abstract）

我们提出 Galileo——一个**高度多模态的 Transformer 模型**，用于表示多种遥感数据模态：多光谱光学影像、合成孔径雷达（SAR）、高程、天气、伪标签等，跨越空间和时间维度。这些输入对多样化的遥感任务（如作物制图、洪水检测）至关重要。然而，学习遥感数据的共享表示极具挑战性，因为相关数据模态种类繁多，且目标物体的尺度差异巨大——从小型船只（1-2像素，快速变化）到冰川（数千像素，缓慢变化）。

我们提出一种**新颖的自监督学习（SSL）算法**，通过掩码建模，从灵活的多模态输入集合中提取**多尺度特征**。我们的**双目标对比学习损失**（全局Global + 局部Local）在**预测目标深度**（深层表征 vs. 浅层输入投影）和**掩码策略**（结构化 vs. 非结构化）上有本质区别。

**Galileo 是一个通用模型（single generalist model），在 11 个基准、多种任务上，超越了针对卫星图像和像素时序数据的 SoTA 专用模型。**

## 1. 引言（Introduction）

* 遥感+机器学习在粮食安全、灾害响应等社会重要领域影响日益增长，但标注数据昂贵且难以获取。
* 自监督学习（SSL）可以开发利用大量未标注遥感数据。

**Galileo 的三个核心创新：**
1. **新SSL算法** — 扩展掩码数据建模框架，同时学习"局部（local）"和"全局（global）"特征
2. **全球采样的多模态预训练数据集** — 127,155个样本，覆盖9种遥感数据模态
3. **统一模型架构** — 单一模型同时处理单时相影像、多时相影像、像素时序数据

**关键结果：** Galileo-Base 排名超过更大的专用遥感模型（SatMAE, CROMA），且同一组权重在像素时序任务上也超过 Presto。

## 2. 方法：全局、局部、多模态自监督

### 2.1 核心思想

遥感应用需要两种互补的能力：
- **全局特征（Global）** → 适合分类任务（区分不同地块的作物类型）
- **局部特征（Local）** → 适合分割任务（精确描绘地块边界）

Galileo 的 SSL 同时训练这两个目标：

```
L_Galileo = ½(L_Global + L_Local)
```

### 2.2 输入 Token 化

输入数据根据时空特性分为四类：
| 类型 | 维度变换 | 示例 |
|------|---------|------|
| 时空数据（Space-Time） | (H,W,T,C) → (H/P·W/P·T·G) × D | Sentinel-2 时序 |
| 空间数据（Space） | (H,W,C) → (H/P·W/P·G) × D | 单时相影像 |
| 时间数据（Time） | (T,C) → (T·G) × D | 像素时序 |
| 静态数据（Static） | (C) → G × D | 高程、人口 |

> 其中 G 是通道组（channel groups），将相关的光谱/数据通道分为一组。

添加四种位置编码：
- **空间正弦位置编码** — 编码像素空间位置
- **时间正弦位置编码** — 编码采集时间
- **可学习通道嵌入** — 区分不同数据模态
- **月份嵌入** — 编码季节性规律

### 2.3 Galileo 自监督学习框架

采用**潜在预测框架（Latent Prediction Framework）**：

```
1. 从数据集中采样，创建两个不同的"视图" x1 和 x2
2. "在线"编码器（Online Encoder）提取可见token的特征 z1 = E(x1)
3. "目标"编码器（Target Encoder，EMA更新）提取目标token特征 z2 = E_EMA(x2)
4. 预测器（Predictor）用目标视图的位置嵌入作为"占位查询"，交叉注意力到z1，预测 p
5. 对比损失 L(p, z2) 更新在线编码器
```

#### 全局目标（Global Objective）— 适合分类

- **掩码策略：** 结构化掩码（空间掩码或时间掩码），跨通道组随机选择
- **目标深度：** 使用目标编码器的**不同深度层**计算目标（深层语义特征）
- **对比损失 PatchDiscB：** 在批次内采样负样本，使不同输入之间的表示更具区分性

```
L_global = PatchDiscB(  P(e2, E(x1)),  sg(E^ℓ_EMA(x2))  )
```

#### 局部目标（Local Objective）— 适合分割

- **掩码策略：** 非结构化随机掩码（5%可见，50%目标）
- **目标深度：** 仅使用目标编码器的**线性投影层**（浅层特征，深度=0）
- **对比损失 PatchDisc：** 仅在同一输入内区分不同patch

```
L_local = PatchDisc(  P(e2, E(x1)),  sg(E^proj_EMA(x2))  )
```

#### 关键设计对比

| 设计维度 | 全局任务 | 局部任务 |
|---------|---------|---------|
| 掩码策略 | 结构化（space/time） | 非结构化随机 |
| 预测目标深度 | 深层（varied depths） | 浅层（depth=0，线性投影） |
| 对比损失 | PatchDiscB（跨样本负例） | PatchDisc（样本内） |
| 适合任务 | 分类 | 分割 |

#### 消融实验验证

* 全局任务促进**输入间**特征多样性（between-input diversity = 0.25）
* 局部任务促进**输入内**特征多样性（within-input diversity = 0.34）
* 联合训练同时保留两者的优势

### 2.4 预训练数据

- **127,155 个训练点**，全球范围采样（使用 H3 六边形索引）
- **采样策略：** 使用 WorldCover 地物类别 + 经纬度做 k-means 聚类（k=150,000），确保地理和语义多样性
- **9 种数据模态：**

| 模态 | 类型 | 说明 |
|------|------|------|
| Sentinel-2 RGB | 时空 | 10m光学RGB |
| Sentinel-2 SWIR | 时空 | 短波红外 |
| Sentinel-2 Red-Edge | 时空 | 红边波段 |
| Sentinel-2 NIR (10m/20m) | 时空 | 近红外 |
| Sentinel-2 NDVI | 时空 | 植被指数 |
| Sentinel-1 SAR | 时空 | 合成孔径雷达（升轨+降轨） |
| SRTM 高程 | 空间 | 30m DEM |
| ERA5 天气 | 时间 | 温度、降水等 |
| TerraClimate | 时间 | 月度气候数据 |
| VIIRS 夜光 | 时间 | 夜间灯光 |
| Dynamic World | 空间 | 土地覆盖分类图（伪标签） |
| World Cereal | 空间 | 谷物种植图（伪标签） |
| LandScan 人口 | 静态 | 人口分布 |
| 地理位置 | 静态 | 经纬度编码 |

## 3. 评估设置

### 评估方法
* **kNN 探测（kNN probing）：** 冻结编码器 → 用训练集特征做 k=20 最近邻分类
* **线性探测（Linear probing）：** 冻结编码器 → 训练线性分类头
* **微调（Finetuning）：** 全模型微调

### 训练集比例
* 100%, 20%, 5%, 1% — 测试不同标注量下的性能

### 评估数据集（11个基准）

| 数据集 | 任务类型 | 说明 |
|--------|---------|------|
| m-EuroSat | 图像分类 | 土地覆盖分类，64×64，2000张 |
| m-BigEarthNet | 图像分类（多标签） | 120×120，20000张 |
| m-So2Sat | 图像分类 | 城市气候区，32×32，19992张 |
| m-Brick-Kiln | 图像分类 | 砖窑检测，64×64 |
| m-Cashew-Plant | 图像分割 | 腰果种植园，256×256 |
| m-SA-Crop-Type | 图像分割 | 南非作物类型，256×256 |
| MADOS | 图像分割 | 海洋垃圾检测，140×140 |
| Sen1Floods11 | 图像分割 | SAR洪水检测 |
| **PASTIS** | **时序分割** | **作物地块分割，128×128，38-61期** |

## 4. 主要结果

### 4.1 综合排名（Table 1）

**Galileo-Base 是所有模型中表现最好的：**
- 图像任务排名：3.0（第1）
- 像素时序任务排名：1.8（第1）
- 远超图像专用模型（SatMAE, CROMA）和时序专用模型（Presto）

Galileo 是**唯一同时支持多光谱、SAR、额外模态、空间维度、多时相（>1期和>4期）**的模型。

### 4.2 图像分类（Tables 3-4）

**kNN 线性探测：**
- EuroSat 100%: Galileo-Base 93.0%，排名第1
- EuroSat 1%: Galileo-Base 56.6%，排名第1（少样本优势尤为明显）

**微调：**
- EuroSat 100%: Galileo-Base 97.7%，排名第1
- So2Sat 100%: Galileo-Base 63.3%，排名第1

### 4.3 图像分割（Table 5）

**线性探测 mIoU：**
- m-Cashew-Plant 100%: Galileo-Base 33.0%，排名第1
- m-SA-Crop-Type 100%: Galileo-Base 30.1%，排名第2
- MADOS 100%: Galileo-Base 67.6%，排名第1
- Sen1Floods11 100%: Galileo-Base 79.4%，排名第1
- **PASTIS 100%: Galileo-Base 39.2%，排名第2**（仅次于 CROMA-Base 44.4%）

### 4.4 像素时序分类（Table 6）

**线性探测：**
- Galileo-Base 在所有四个数据集上平均排名第1
- 超越了时序专用模型 Presto

### 4.5 效率 vs. 精度（Figure 4）

Galileo 通过调整 patch size（4/8/16）和模型大小（Nano/Tiny/Base），可在精度和推理成本之间灵活权衡。

## 5. 消融实验（Ablations）

### 5.1 全局任务消融（Table 7）
* **结构化掩码（space+time）** 对全局特征提取至关重要
* 使用深层目标（varied depths）优于固定深度
* Masked MSE 重建损失不如 PatchDiscB 对比损失

### 5.2 局部任务消融（Table 8）
* **浅层目标（depth=0）** 对分割任务极其有效（MADOS 上 71.5 vs. 58.9 mIoU）
* 非结构化随机掩码优于结构化掩码 9 个点（MADOS）
* 对比损失优于 MSE 重建损失 5+ 个点

### 5.3 联合训练消融（Table 9）
* 双目标联合训练**比单目标更稳定**
* 联合训练在 EuroSat 上 100% 运行超过 80%，而单目标仅 63%

### 5.4 数据模态消融（Table 11）
* 移除 TerraClimate → EuroSat 下降 9.8%，CropHarvest 下降 5.1%
* 移除 VIIRS（夜光）→ EuroSat 下降 9.9%
* 移除 LandScan（人口）→ MADOS 下降 7.1%
* 每种模态都对某些下游任务有显著贡献

## 6. 相关工作（Related Work）

Galileo 的自监督学习设计融合了多条技术路线：

| 范式 | 代表性工作 | 在RS中的应用 |
|------|-----------|-------------|
| MAE 风格（像素重建） | SatMAE(++), CROMA, MMEarth, DOFA | 预测被遮挡像素 |
| I-JEPA 风格（潜在预测） | AnySat | 预测被遮挡patch的表征 |
| SimCLR/Barlow Twins（对比学习） | CROMA, DeCUR, AnySat | 对比不同增强视图 |
| LatentMIM（潜在掩码对比） | 之前无RS实例 | 在潜在空间中对比patch |
| **Galileo（本文）** | **本文** | **同时在不同深度吸引同类patch、排斥异类patch** |

**与 AnySat 的关键区别：**
- AnySat 缺失 Galileo 使用的许多额外模态（天气、夜光、人口、伪标签等）
- 这些额外模态在实际应用中被证明至关重要

## 8. 对项目的启示与可借鉴之处

**这篇论文与你的项目高度相关！以下是具体的参考价值：**

### 8.1 方法层面

1. **"Global + Local" 双目标SSL框架**
   - 你的耕地分割项目同时需要分类能力（区分作物类型）和分割能力（描绘地块边界）
   - Galileo 的 Local 目标专门为分割任务设计，可以直接参考

2. **"Deep-Shallow" 对比学习**
   - 全局任务用深层编码 → 适合分类
   - 局部任务用浅层投影 → 适合分割
   - 这是一个非常巧妙的差异化设计

3. **时序融合设计**
   - Galileo 灵活处理空间/时间维度，可以处理像素时序、多时相图像、单时相图像
   - 你的 PASTIS 数据集是典型的多时相图像（T × C × H × W），可以参考 Galileo 的 token 化方式

### 8.2 数据层面

1. **多模态预训练数据**
   - Galileo 证明了加入额外模态（天气、高程、SAR等）可以显著提升下游任务
   - 你的项目可以考虑在PASTIS之外接入更多数据源（如 Sentinel-1 SAR、SRTM 高程）

2. **全球多样性采样**
   - 使用 k-means 聚类确保地理和语义多样性
   - 这个采样策略可以直接用到你的预训练数据构建中

### 8.3 评估层面

1. **多训练集比例评估**
   - 论文在 100%, 20%, 5%, 1% 四种标注量下评估
   - 这是评估预训练模型"少样本学习能力"的标准做法
   - **你的实验报告也应该采用这种评估方式**

2. **PASTIS 上的 baselines**
   - Galileo-Base 在 PASTIS 上 linear probing 达 39.2% mIoU
   - CROMA-Base 达 44.4% mIoU（使用的是多时相融合）
   - **这些是你需要对比/超越的基线**

### 8.4 架构层面

1. **ViT 作为统一backbone**
   - 多种模型规模可选（Nano/Tiny/Base = 0.8M/5.3M/85M 参数）
   - 你的项目也可以从轻量模型开始，逐步扩大

2. **Patch Size 很重要**
   - Galileo 显示 patch size 4 对分割任务更好，patch size 8/16 对分类任务也可接受
   - PASTIS 的 128×128 图像用 patch size=4 可以得到 32×32 的 token grid

## 9. 总结

**Galileo 不是你可以直接调用的现成预训练模型**——它的主要贡献是一个新的 SSL 算法框架和多模态预训练数据集方案。

**对项目的具体帮助：**

| 方面 | 可以直接用的 | 需要自己实现的 |
|------|------------|--------------|
| 数据 | PASTIS dataloader（已有） | 多模态数据收集与对齐 |
| 模型 | SMP 预训练encoder（已有）| Galileo-style 时序token化 + 融合 |
| 训练 | 训练框架（已有） | Galileo-style 双目标SSL预训练 |
| 评估 | 多比例评估方案 ✓ | — |
| 对比 | Galileo/CROMA/Presto baseline数字 ✓ | — |

**最重要的启示：** 对于耕地分割这个同时涉及分类和分割的任务，"全局+局部"双目标预训练是一个值得探索的方向。你可以在 SMP 预训练 encoder 的基础上，设计类似的 SSL 预训练阶段，在 PASTIS 数据上学习更好的时序-空间特征。
